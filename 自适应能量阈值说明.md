# 自适应能量阈值说明

## 功能概述

自适应能量阈值是一个智能算法，用于自动检测音乐中的高潮部分（如鼓点、强节奏），只在这些关键时刻生成粒子效果，让可视化更具表现力和节奏感。

## 工作原理

### 1. 能量历史追踪
- 维护一个滑动窗口，记录最近 N 帧的能量值（默认90帧，约3秒）
- 实时分析能量的变化趋势和统计特征

### 2. 多维度阈值计算

算法综合使用三种方法计算动态阈值，并取中位数：

#### 方法1：统计阈值
```
阈值 = 均值 + 0.5 × 标准差
```
- 捕捉高于平均水平的能量峰值
- 适应音乐的整体响度

#### 方法2：最大值百分比
```
阈值 = 历史最大值 × 40%
```
- 避免过于敏感，只在真正的高潮触发
- 防止持续生成粒子

#### 方法3：移动平均倍数
```
阈值 = 近期平均值 × 峰值检测比例(默认1.5)
```
- 检测相对于近期的能量突增
- 更敏感地捕捉节奏变化

### 3. 峰值增强检测

除了基本的阈值比较，还包括：

- **能量突增检测**：如果当前能量是前5帧平均的1.5倍以上，也视为峰值
- **相对判断**：能量必须同时高于全局平均值
- 这能更准确地捕捉鼓点等瞬时高能量事件

### 4. 阈值限制

为避免极端情况，阈值被限制在合理范围内：
- 最小值：0.01（避免过于敏感）
- 最大值：0.2（确保有粒子生成）

## 配置参数

在 `config.py` 中可以调整以下参数：

### `USE_ADAPTIVE_THRESHOLD`
- **类型**：布尔值
- **默认**：`True`
- **说明**：是否启用自适应阈值
  - `True`：自动适应音乐，推荐
  - `False`：使用固定阈值 `ENERGY_THRESHOLD`

### `ENERGY_THRESHOLD`
- **类型**：浮点数
- **默认**：`0.02`
- **说明**：固定能量阈值，仅在 `USE_ADAPTIVE_THRESHOLD=False` 时使用

### `ADAPTIVE_HISTORY_SIZE`
- **类型**：整数（帧数）
- **默认**：`90`（@30fps ≈ 3秒）
- **建议范围**：60-120
- **说明**：能量历史窗口大小
  - 更小：对短期变化更敏感，适合节奏快的音乐
  - 更大：更稳定，适合节奏慢或变化大的音乐

### `PEAK_DETECTION_RATIO`
- **类型**：浮点数
- **默认**：`1.5`
- **建议范围**：1.3-2.0
- **说明**：峰值检测倍数
  - 更小（如1.3）：更频繁触发，粒子更多
  - 更大（如2.0）：只在明显峰值触发，粒子更少

### `ADAPTIVE_MIN_THRESHOLD`
- **类型**：浮点数
- **默认**：`0.01`
- **说明**：自适应阈值的最小值，防止过于敏感

### `ADAPTIVE_MAX_THRESHOLD`
- **类型**：浮点数
- **默认**：`0.2`
- **说明**：自适应阈值的最大值，确保能生成粒子

## 使用场景

### 适合自适应阈值的音乐类型
- ✅ **电子音乐**：明确的鼓点和节奏
- ✅ **流行音乐**：清晰的节拍和高潮部分
- ✅ **摇滚音乐**：强烈的鼓点和节奏变化
- ✅ **Hip-Hop/Rap**：突出的低音和节奏

### 可能需要调整的情况
- **古典音乐**：动态范围大，可能需要更大的 `ADAPTIVE_HISTORY_SIZE`（120-180）
- **环境音乐**：能量变化小，可能需要降低 `PEAK_DETECTION_RATIO`（1.2-1.3）
- **重金属**：持续高能量，可能需要更大的 `PEAK_DETECTION_RATIO`（1.8-2.0）

## 调试信息

运行时，控制台会每100帧输出一次调试信息：

```
新生成: 15个, 屏幕总数: 342个, 能量: 0.085, 阈值: 0.052, 提前: 83帧
```

解读：
- **新生成**：本帧新生成的粒子数量
- **屏幕总数**：当前屏幕上存活的粒子总数
- **能量**：当前帧的能量值
- **阈值**：当前的动态阈值
- **提前**：预测性同步的帧数（仅启用时显示）

观察规律：
- 如果 `能量 > 阈值`，会生成粒子
- 如果粒子太多，增大 `PEAK_DETECTION_RATIO`
- 如果粒子太少，减小 `PEAK_DETECTION_RATIO`

## 调优建议

### 步骤1：观察默认效果
先使用默认配置运行，观察粒子生成的频率和时机。

### 步骤2：根据音乐类型调整

**粒子太多（过于频繁）：**
```python
PEAK_DETECTION_RATIO = 1.8  # 提高阈值
ADAPTIVE_HISTORY_SIZE = 120  # 增加历史窗口
```

**粒子太少（不够频繁）：**
```python
PEAK_DETECTION_RATIO = 1.3  # 降低阈值
ADAPTIVE_HISTORY_SIZE = 60  # 减小历史窗口
```

**粒子不同步（在安静处生成）：**
```python
ADAPTIVE_MIN_THRESHOLD = 0.02  # 提高最小阈值
```

### 步骤3：微调粒子数量

如果生成时机正确，但数量不合适，调整：
```python
PARTICLE_SPAWN_RATE = 55.0  # 每单位能量生成的粒子数
```

## 技术细节

### 算法复杂度
- **时间复杂度**：O(n)，其中 n 是历史窗口大小
- **空间复杂度**：O(n)
- **性能影响**：几乎可以忽略（<1ms per frame）

### 冷启动处理
- 前10帧使用固定阈值 `ENERGY_THRESHOLD`
- 之后逐步建立历史数据，切换到自适应模式
- 确保视频开始时就能正常工作

### 与预测性同步的配合
- 自适应阈值分析的是"未来帧"的能量（如果启用预测性同步）
- 这确保了粒子不仅在正确的时机生成，而且能在正确的时间到达圆环
- 实现了视觉效果与音乐的完美同步

## 示例配置

### 配置1：适合大多数流行音乐（默认）
```python
USE_ADAPTIVE_THRESHOLD = True
ADAPTIVE_HISTORY_SIZE = 90
PEAK_DETECTION_RATIO = 1.5
ADAPTIVE_MIN_THRESHOLD = 0.01
ADAPTIVE_MAX_THRESHOLD = 0.2
```

### 配置2：适合节奏强烈的电子音乐
```python
USE_ADAPTIVE_THRESHOLD = True
ADAPTIVE_HISTORY_SIZE = 60  # 更短的窗口，快速响应
PEAK_DETECTION_RATIO = 1.3  # 更低的阈值，更多粒子
ADAPTIVE_MIN_THRESHOLD = 0.015
ADAPTIVE_MAX_THRESHOLD = 0.15
```

### 配置3：适合古典音乐或环境音乐
```python
USE_ADAPTIVE_THRESHOLD = True
ADAPTIVE_HISTORY_SIZE = 120  # 更长的窗口，平滑变化
PEAK_DETECTION_RATIO = 1.8  # 更高的阈值，只在明显高潮
ADAPTIVE_MIN_THRESHOLD = 0.005
ADAPTIVE_MAX_THRESHOLD = 0.25
```

### 配置4：关闭自适应（使用固定阈值）
```python
USE_ADAPTIVE_THRESHOLD = False
ENERGY_THRESHOLD = 0.02  # 手动设定固定阈值
```

## 常见问题

### Q: 自适应阈值会影响性能吗？
A: 几乎不会。计算开销极小（<1ms），相比视频编码可以忽略。

### Q: 为什么有时粒子在安静处也会生成？
A: 可能是历史窗口还在适应中，或者该音乐的动态范围很小。可以提高 `ADAPTIVE_MIN_THRESHOLD`。

### Q: 可以完全手动控制阈值吗？
A: 可以，设置 `USE_ADAPTIVE_THRESHOLD = False`，然后调整 `ENERGY_THRESHOLD`。

### Q: 如何判断当前配置是否合适？
A: 观察调试输出，如果在鼓点处 `能量` 明显高于 `阈值`，配置就是合理的。

## 总结

自适应能量阈值算法通过实时分析音频能量的统计特征和变化趋势，智能地决定何时生成粒子，让可视化效果与音乐的节奏和情感完美契合。这是一个"设置后就忘记"的功能，大部分情况下默认配置即可获得出色效果。
