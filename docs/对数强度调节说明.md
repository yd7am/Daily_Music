# 对数强度调节说明

## 问题：低频分辨率看起来小于高频

### 原因

完全对数分组时：
```
条1:  20-30 Hz      (10 Hz 宽)   ← 视觉上看起来"窄"
条2:  30-45 Hz      (15 Hz 宽)
条3:  45-70 Hz      (25 Hz 宽)
...
条58: 8000-13000 Hz (5000 Hz 宽) ← 视觉上看起来"宽"
条59: 13000-18000 Hz (5000 Hz 宽)
条60: 18000-20000 Hz (2000 Hz 宽)
```

虽然低频每个条代表更小的频率范围（更精确），但**视觉上看起来分辨率低**。

---

## ✅ 解决方案：可调节的对数强度

在 `config.py` 中新增参数：

```python
LOG_FREQ_POWER = 0.5  # 对数强度
```

### 参数说明

| 值 | 效果 | 低频分辨率 | 高频分辨率 | 视觉均匀度 |
|----|------|----------|----------|----------|
| **0.0** | 完全线性 | ████░░░░ 差 | ████░░░░ 差 | ████████ 完美 |
| **0.3** | 接近线性 | ██████░░ 较好 | ██████░░ 较好 | ███████░ 很好 |
| **0.5** | 温和对数 | ████████ 好 | ██████░░ 适中 | ██████░░ 好 ✅ |
| **0.7** | 较强对数 | ██████████ 很好 | ████░░░░ 较差 | ████░░░░ 一般 |
| **1.0** | 完全对数 | ██████████ 完美 | ██░░░░░░ 差 | ██░░░░░░ 差 |

## 🔬 技术原理

### 原始对数分组（power = 1.0）

```python
# 完全对数
log_freqs = np.logspace(log10(20), log10(20000), 60)
# 结果：[20, 25, 32, 40, 50, ..., 15849, 20000]
```

### 新的可调节方法（power = 0.5）

```python
# 1. 线性空间
linear = [0, 0.017, 0.033, ..., 1.0]

# 2. 应用幂函数
adjusted = linear ^ 0.5
# [0, 0.13, 0.18, ..., 1.0]

# 3. 映射到对数频率
log_freqs = 10^(log_min + adjusted * (log_max - log_min))
```

**效果**：
- power = 0: 相当于线性
- power = 0.5: 介于线性和对数之间
- power = 1.0: 完全对数

## 📝 总结

**问题**：低频分辨率看起来小于高频  
**原因**：完全对数分组导致的视觉差异  
**解决**：添加 `LOG_FREQ_POWER` 参数调节对数强度

**推荐值**：
- **0.5** - 大多数情况的最佳平衡 ✅
- **0.4** - 追求更均匀的视觉效果
- **0.6** - 保留更多低频细节

**配合使用**：
- 增加 `NUM_BARS` 到 100-120
- 调整 `LOW/HIGH_FREQ_WEIGHT` 平衡显示

这样可以得到既好看又保留音乐特性的频谱！🎵
