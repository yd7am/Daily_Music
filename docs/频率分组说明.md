# 频率分组说明

## 为什么低频"分辨率"看起来小于高频？

### 当前使用：对数分组

```python
log_freqs = np.logspace(20, 20000, 60)
# 结果：
# 条1: 20-30 Hz      (10 Hz 宽)    ← 窄
# 条2: 30-45 Hz      (15 Hz 宽)
# ...
# 条30: 500-1000 Hz  (500 Hz 宽)
# ...
# 条60: 10000-20000 Hz (10000 Hz 宽) ← 很宽
```

**特点**：
- ✅ 符合人耳听觉特性（对数感知）
- ✅ 低频细节更多（贝斯、鼓）
- ❌ 视觉上低频"分辨率"看起来低

**为什么这样设计**？
人耳对频率的感知是对数的：
- 20Hz → 40Hz 听起来差别很大
- 10000Hz → 10020Hz 几乎听不出差别

---

## 💡 解决方案

### 方案 1：使用线性分组（均匀分辨率）

每个条形代表相同宽度的频率范围。

#### 实现方法

在 `audio_visualizer.py` 中修改：

```python
def _compute_spectrogram(self):
    """计算音频频谱图"""
    # ... 现有代码 ...
    
    # 将对数分组改为线性分组
    self.freq_bins = self._get_linear_freq_bins()  # 改这里

def _get_linear_freq_bins(self):
    """获取线性频率分组（均匀分辨率）"""
    # 线性等分频率范围
    return np.linspace(0, len(self.freq_indices), 
                      self.config['num_bars'] + 1, dtype=int)
```

**效果**：
```
条1:  20-350 Hz      (330 Hz 宽)  ← 每个条形
条2:  350-680 Hz     (330 Hz 宽)     宽度相同
条3:  680-1010 Hz    (330 Hz 宽)
...
条60: 19670-20000 Hz (330 Hz 宽)
```

**优点**：
- ✅ 视觉上每个条形代表相同分辨率
- ✅ 看起来更"均匀"

**缺点**：
- ❌ 低频细节少（20-350Hz 全挤在一个条里）
- ❌ 高频过度细分（没必要）

---

### 方案 2：折中方案（混合分组）

低频用线性，高频用对数。

```python
def _get_hybrid_freq_bins(self):
    """混合频率分组"""
    # 0-2000 Hz: 线性分组（40个条）
    # 2000-20000 Hz: 对数分组（20个条）
    
    low_bins = int(self.config['num_bars'] * 0.7)  # 70% 给低频
    high_bins = self.config['num_bars'] - low_bins
    
    # 低频部分（线性）
    low_freq_mask = self.freqs <= 2000
    low_indices = np.where(low_freq_mask)[0]
    low_bins_array = np.linspace(0, len(low_indices), 
                                 low_bins + 1, dtype=int)
    
    # 高频部分（对数）
    high_freq_mask = self.freqs > 2000
    high_indices = np.where(high_freq_mask)[0]
    log_high = np.logspace(0, np.log10(len(high_indices)), 
                          high_bins + 1)
    high_bins_array = (log_high.astype(int) + len(low_indices))
    
    # 合并
    return np.concatenate([low_bins_array[:-1], high_bins_array])
```

## 🎛️ 三种分组对比

### 对数分组（当前）✅

```
低频细节：  ████████████ 很多
高频细节：  ██░░░░░░░░░░ 少
视觉分辨率：低频低，高频高
```

**适合**：
- 古典音乐
- 人声
- 需要低频细节的音乐

### 线性分组

```
低频细节：  ████░░░░░░░░ 少
高频细节：  ████░░░░░░░░ 少
视觉分辨率：完全均匀
```

**适合**：
- 视觉美观优先
- 不关心频率准确性

### 混合分组

```
低频细节：  ████████░░░░ 较多
高频细节：  ████░░░░░░░░ 适中
视觉分辨率：相对均匀
```

**适合**：
- 电子音乐
- EDM
- 平衡视觉和准确性

## 🔧 快速切换

### 切换到线性分组

1. 打开 `audio_visualizer.py`
2. 找到第 82 行：
   ```python
   self.freq_bins = self._get_log_freq_bins()
   ```
3. 改为：
   ```python
   self.freq_bins = np.linspace(0, len(self.freq_indices), 
                                self.config['num_bars'] + 1, dtype=int)
   ```

### 增加条形数量（提高分辨率）

在 `config.py` 中：
```python
NUM_BARS = 120  # 从 60 改为 120（翻倍）
```

**效果**：
- 整体分辨率提高
- 低频和高频都更细腻
- 但计算量增加

## 📊 实际频率分布示例

### 对数分组（60条）

```
条1-10:   20-200 Hz      (贝斯、鼓)
条11-20:  200-500 Hz     (男声基频)
条21-30:  500-1000 Hz    (女声基频)
条31-40:  1000-2500 Hz   (人声高音)
条41-50:  2500-6000 Hz   (镲片、气音)
条51-60:  6000-20000 Hz  (极高频、泛音)
```

### 线性分组（60条）

```
条1-10:   20-3300 Hz     (所有人声都在这里！)
条11-20:  3300-6600 Hz
条21-30:  6600-10000 Hz
条31-40:  10000-13300 Hz (大部分人听不到)
条41-50:  13300-16600 Hz (基本没内容)
条51-60:  16600-20000 Hz (完全没内容)
```

可以看出，线性分组会把大量重要的低频信息挤在少数几个条里。

## 💡 推荐配置

### 推荐 1：保持对数分组 + 增加条数

```python
# config.py
NUM_BARS = 100  # 或 120
```

这样低频分辨率会提高，视觉上更均匀。

### 推荐 2：对数分组 + 调整频率范围

```python
# config.py
FREQ_MIN = 30      # 从 20 改为 30（忽略极低频）
FREQ_MAX = 16000   # 从 20000 改为 16000（忽略人耳听不到的）
NUM_BARS = 80
```

这样中频分辨率更高。

## 🎵 总结

**为什么低频"分辨率"低？**
- 使用了对数分组
- 这是有意为之，符合听觉特性
- 低频的每个条代表更窄的频率范围
- 但视觉上看起来"宽"

**如何改善？**
1. 增加条形总数（推荐 80-120）
2. 调整频率范围（去掉极低和极高频）
3. 如果追求视觉均匀，改用线性分组

**我的建议**：
保持对数分组，但增加条数到 100-120，这样既保持了音乐特性，视觉上也更细腻！
