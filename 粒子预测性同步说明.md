# 粒子预测性同步说明

## 🎯 问题描述

### 延迟问题

```
圆心到圆上的距离：250 像素
粒子速度：3 像素/帧
到达时间：250 / 3 ≈ 83 帧

在 30fps 视频中：83 / 30 ≈ 2.8 秒！
```

**结果**：粒子到达圆上时，频谱已经变化了，不同步！

### 期望效果

```
粒子到达圆上时 = 频谱显示该能量

需要：提前生成粒子
```

---

## ✅ 解决方案：预测性同步

### 核心思想

**在当前帧使用未来帧的能量来生成粒子**

```python
# 计算提前量
travel_frames = CIRCLE_RADIUS / PARTICLE_SPEED
# = 250 / 3 = 83 帧

# 获取未来帧的能量
future_frame_idx = current_frame_idx + travel_frames
future_energy = get_energy(future_frame_idx)

# 用未来的能量生成粒子
spawn_particles(future_energy)
```

### 时间线示意图


### 数学推导

```
粒子位置 = 起点 + 速度 × 时间

到达圆上时：
距离 = CIRCLE_RADIUS
时间 = 距离 / 速度
     = 250 / 3
     = 83.33 帧

提前量 = 83 帧（向下取整）
```

## 🎛️ 配置参数

### 在 config.py 中

```python
PARTICLE_PREDICTION = True   # 启用预测性同步 ✅
PARTICLE_SPEED = 3.0         # 粒子速度（影响提前量）
CIRCLE_RADIUS = 250          # 圆半径（影响提前量）
```

### 提前量计算

```python
提前帧数 = int(CIRCLE_RADIUS / PARTICLE_SPEED)

例如：
CIRCLE_RADIUS = 250
PARTICLE_SPEED = 3.0
→ 提前量 = 250 / 3 = 83 帧

PARTICLE_SPEED = 5.0
→ 提前量 = 250 / 5 = 50 帧
```

## 💡 代码实现

### 关键代码

```python
# audio_visualizer.py

# 1. 计算提前量
travel_frames = int(config.CIRCLE_RADIUS / config.PARTICLE_SPEED)
# = 250 / 3 = 83 帧

# 2. 计算未来帧索引
future_frame_idx = current_frame_idx + travel_frames

# 3. 获取未来帧的振幅
future_amplitudes = self._get_frame_amplitudes(future_frame_idx)

# 4. 计算未来帧的能量
future_energy = np.mean(future_amplitudes)

# 5. 用未来的能量生成粒子
spawn_particles(future_energy)
```

### 自动处理边界

```python
# _get_frame_amplitudes 方法会自动处理超出范围
if frame_idx >= self.db.shape[1]:
    frame_idx = self.db.shape[1] - 1  # 使用最后一帧
```

## 🔧 调优建议

### 如果粒子到达时还是不同步

#### 检查 1：速度是否正确

```python
# 手动测量实际速度
实际提前量 = 到达时间 / 30  # 秒
应有提前帧数 = 实际提前量 × 30

检查是否 = CIRCLE_RADIUS / PARTICLE_SPEED
```

#### 检查 2：圆半径是否正确

```python
# 确认 CIRCLE_RADIUS 与实际绘制的半径一致
CIRCLE_RADIUS = 250  # 必须准确
```

#### 检查 3：帧率是否稳定

```python
# 如果视频帧率不是 30fps
VIDEO_FPS = 25  # 假设
提前时间 = 83 / 25 = 3.3 秒（而不是 2.8 秒）
```

### 微调提前量

如果需要手动调整：

```python
# config.py
PARTICLE_PREDICTION_OFFSET = 0  # 额外的帧偏移（可正可负）

# audio_visualizer.py
future_frame_idx = frame_idx + travel_frames + config.PARTICLE_PREDICTION_OFFSET
```

## 🎵 应用场景

### 适用情况

- ✅ 粒子速度较慢（< 5 px/帧）
- ✅ 圆半径较大（> 200 px）
- ✅ 需要精确同步视觉效果
- ✅ 圆形频谱显示

### 不适用情况

- ❌ 粒子速度很快（> 10 px/帧）→ 提前量很小，意义不大
- ❌ 条形频谱等其他显示方式
- ❌ 粒子不需要到达特定位置

## 💡 进阶优化

### 方案 1：动态预测

根据粒子实际位置调整：

```python
# 计算粒子当前距离圆心的距离
distance = sqrt((x - center_x)^2 + (y - center_y)^2)

# 根据距离动态获取能量
remaining_frames = (CIRCLE_RADIUS - distance) / PARTICLE_SPEED
energy = get_energy(current_frame + remaining_frames)
```

### 方案 2：插值预测

使用插值提高精度：

```python
# 非整数帧的插值
travel_frames = CIRCLE_RADIUS / PARTICLE_SPEED  # 83.33
frame1 = int(travel_frames)  # 83
frame2 = frame1 + 1  # 84
ratio = travel_frames - frame1  # 0.33

# 插值能量
energy = (1 - ratio) * get_energy(frame1) + ratio * get_energy(frame2)
```
